# 2026-04-28 更新记录（UI 与预览）

本文档记录本轮围绕前端可用性和预览稳定性的更新。

## 1. 双语 UI 能力

- 新增中英文 UI 切换。
- 默认语言为简体中文。
- 顶部 Header 增加语言选择器。

## 2. HTML 创意预览：样式导入能力

- 新增 `HTML样式导入` 面板。
- 支持导入外部样式（Text/JSON 两种格式）。
- 导入字段要求：
  - `id`
  - `name-en`
  - `name-zh`
  - `css`
- 导入后可直接应用，并加入样式下拉。
- 样式下拉显示中英文双名称（例如：`现代深色 (Modern Dark)`）。

## 3. HTML 样式规范（给 AI 用）

- 新增规范文档：`frontend/public/HTML_STYLE_SPEC.md`（会构建到 `public/HTML_STYLE_SPEC.md`）。
- 规范弹窗支持：
  - 复制规范
  - 在线编辑
  - 保存编辑（保存到浏览器本地）
  - 恢复默认规范

## 4. HTML 样式二次编辑与管理

- 新增“当前样式”操作：
  - 复制当前样式（导出为可再次导入的文本格式）
  - 加载当前样式到编辑区
  - 删除当前样式
- 删除策略：
  - 仅允许删除自定义样式
  - 内置预设不可删除

## 5. Word 快速预览报错修复

- 修复错误：`Invalid character in header content ["Content-Disposition"]`
- 修复方式：
  - 统一处理下载文件名，清理非法字符
  - 使用 ASCII 回退 `filename`
  - 同时输出 `filename*`（UTF-8）

## 6. 预览提示刷屏修复

- 修复“本地预览已加载”频繁刷屏。
- 自动预览触发时静默，不再重复弹成功提示。
- 增加 toast 去重窗口，抑制短时间重复文案。

## 7. 预览切换残留修复

- 修复切换预览模式时旧结果残留。
- 离开模式时释放对应 URL 并清空状态：
  - local / pdf / collabora 分别清理。

## 8. 字体设置可编辑性修复

- 修复“字体输入框删一个字符就失焦”的问题。
- 根因：输入组件在父组件内声明导致反复重建。
- 处理：
  - 输入组件提取为稳定组件。
  - 字体字段改为 `select` 下拉，不再依赖 `datalist`。
- 结果：
  - 点击即可展开字体列表
  - 无需先清空文本
  - 可直接选择切换字体

## 9. 生成器样式写入修复

- 修复字体配置未真正写入 docx 样式的问题。
- 调整 docx 样式对象结构后，字体配置可正确写入 `styles.xml`。
